# Session Context: TS6059 / TS6307 Fix

## Problem
`pnpm nx build aoc-feature` fails with 8 errors (4 TS6059 + 4 TS6307):
- `card-select.modal.ts:13` — `scope.ts` not under rootDir `libs/aoc/feature`
- `section.store.ts:22` — `section-edit.modal.ts` not under rootDir
- `section.store.ts:23` — `message-center.modal.ts` not under rootDir
- `section.store.ts:24` — `card-select.modal.ts` not under rootDir

## Root Cause (confirmed via --traceResolution)
1. `aoc-content.store.ts` in `aoc/feature` imports `SectionEditModal` from `@bk2/cms-section-feature`
2. Nx remaps `@bk2/cms-section-feature` → `dist/libs/cms/section/feature` (correct)
3. TypeScript resolves to `dist/.../index.d.ts`, which re-exports `section.store`
4. `dist/.../section.store.d.ts` now has `import { MatrixChatService } from '@bk2/chat-data-access'` (newly added by user in current working tree changes)
5. `@bk2/chat-data-access` is NOT in `cms-section-feature/package.json`, so Nx does NOT remap it to dist in dependent builds
6. This causes TypeScript to fall back to source resolution, pulling in `section.store.ts` source
7. `section.store.ts` has relative imports to `./section-edit.modal`, `./message-center.modal`, `./card-select.modal` — all outside rootDir of `aoc-feature`

## What Was Already Fixed
- `membership.store.ts`: duplicate local `MEMBERSHIP_I18N_KEYS` block (leftover from i18n migration, referencing undefined `PFX`) was removed. This file is clean.
- Stale Nx cache for `cms-section-feature` was cleared and rebuilt.
- `aoc/feature/tsconfig.lib.json` and `package.json` are at their original state (no added references).

## Two Fixes to Implement

### Option A — Add missing deps to `cms-section-feature/package.json`
File: `libs/cms/section/feature/package.json`
Add to `dependencies`:
- `"@bk2/chat-data-access": "*"`
- `"@capacitor/core": "*"` (also newly imported in section.store.ts)
- `"@capacitor/geolocation": "*"` (also newly imported in section.store.ts)

This ensures Nx remaps these to dist when building dependent libs, so TypeScript uses compiled output instead of source.

### Option B — Remove `SectionStore` from the public barrel
File: `libs/cms/section/feature/src/index.ts`
`SectionStore` (and `section.store`) is an internal implementation detail.
Remove `export * from './lib/section.store'` from `index.ts`.

This breaks the resolution chain: `aoc-feature` never needs to pull in `section.store` types at all.

NOTE: Check whether any external consumers (other libs) actually import `SectionStore` from `@bk2/cms-section-feature` before removing. If yes, Option A is safer.
