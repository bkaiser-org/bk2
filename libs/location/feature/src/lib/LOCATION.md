# Location Domain

## Overview

The Location domain models geographic places (training venues, meeting points, buildings, etc.) that can be referenced by other models — primarily `CalEventModel.locationKey`. Each location stores GPS coordinates, a Google Places ID, an optional what3words address, and altitude/speed/direction metadata for mobile tracking use cases.

## Firestore Collection

Collection name: `locations`

## Field Semantics

| Field | Type | Description |
|---|---|---|
| `bkey` | string | Firestore document ID (stripped on write, re-attached on read) |
| `tenants` | string[] | Multi-tenancy isolation; queries always filter by tenantId |
| `isArchived` | boolean | Soft-delete flag |
| `name` | string | Human-readable name of the location |
| `index` | string | Computed search index string (generated by `getLocationIndex`) |
| `tags` | string | Comma-separated tag list for filtering |
| `address` | string | Formatted address string (from Google Places) |
| `type` | string | Location type category (from `location_type` category list) |
| `latitude` | number | WGS-84 latitude |
| `longitude` | number | WGS-84 longitude |
| `placeId` | string | Google Places ID |
| `what3words` | string | What3words address (e.g. `///index.home.raft`) |
| `seaLevel` | number | Altitude in metres above sea level |
| `speed` | number | Speed in m/s (used for tracking) |
| `direction` | number | Compass direction in degrees |
| `notes` | string | Free-text notes |

## LocationListStore

NgRx Signal Store (provided at component level). State:

| State field | Description |
|---|---|
| `searchTerm` | Text filter on `index` |
| `selectedTag` | Tag chip filter |
| `selectedType` | Location type filter (`'all'` or a specific type) |

Key resource:
- `locationsResource` — backed by `LocationService.list()` (real-time stream from `locations` collection).

Computed:
- `filteredLocations` — applies `nameMatches(index, searchTerm)`, `nameMatches(type, selectedType)`, and `chipMatches(tags, selectedTag)` filters client-side.

Key actions:
- `add(readOnly)` / `edit(location, readOnly)` — open `LocationEditModalComponent` for create/update.
- `delete(location, readOnly)` — calls `LocationService.delete`.
- `showOnMap(location)` — opens `MapViewModalComponent` centred on the location's coordinates.
- `copy(location)` — copies `latitude, longitude` to clipboard with a toast confirmation.

## Components

| Component | Description |
|---|---|
| `LocationListComponent` | Filterable list of locations; delegates CRUD to `LocationListStore` |
| `LocationEditModalComponent` | Full-modal edit form for create/update |
